# Telegram Bot для продажи файлов оптимизации ПК

Рабочий проект Telegram-бота на Python + aiogram 3. Бот автоматически регистрирует пользователей по Telegram ID, показывает личный кабинет, каталог с вложенной структурой, демо-файлы после проверки подписки, покупки через Telegram Stars и CryptoBot, подписки, FAQ, поддержку, отзывы и админ-панель.

## Возможности

- Авторизация по Telegram ID и хранение даты регистрации, баланса и количества покупок.
- Личный кабинет: профиль, история покупок, активные подписки.
- Новый профиль с ID, балансом в Stars/рублях, суммой покупок, суммой пополнений и датой регистрации.
- Пополнение баланса в рублях или Telegram Stars через Telegram Stars и CryptoBot.
- Каталог по схеме: раздел → подкатегория/игра → тип оптимизации → товар.
- Раздел FAQ и визуальные метки товаров: `🔥 Популярное`, `🆕 Новинка`, `💎 Рекомендуем`.
- Демо-версии с обязательной проверкой подписки на канал.
- Защита от повторной бесплатной выдачи демо и повторного начисления одной покупки.
- Покупка через Telegram Stars и CryptoBot.
- Экран оплаты с кнопками `Назад` и `Отмена`, чтобы пользователь не застревал на счёте.
- Промокоды: процент скидки, срок действия, лимит использований, включение, отключение и удаление.
- Отзывы после покупки: пользователь оставляет оценку и комментарий из раздела `Мои покупки`, отзыв уходит владельцам бота в ЛС.
- Экстро-оптимизация с обязательным предупреждением и кнопкой `⚠️ Я понимаю риски`.
- Выдача restore-файла или инструкции для отката экстро-настроек.
- Админ-панель: добавление товара через FSM с кнопками выбора, загрузка фото через Telegram, просмотр товаров, пользователей, продаж и подписок.
- Фото разделов: главное меню, профиль, подписки, поддержка, отзывы, правила, категории и подкатегории.

## Быстрый запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Заполните `.env`:

- `BOT_TOKEN` — токен от `@BotFather`.
- `ADMIN_IDS` — Telegram ID администраторов через запятую.
- `REQUIRED_CHANNEL` — канал для проверки подписки перед выдачей демо.
- `SUPPORT_USERNAME` — username администратора.
- `REVIEWS_CHANNEL_URL` — ссылка на канал отзывов.
- `CRYPTOBOT_TOKEN` — токен CryptoBot, если нужна оплата криптой.

Запуск:

```bash
python main.py
```

## Пополнение баланса

В профиле нажмите `Пополнить баланс`. Пользователь выбирает тип баланса:

- `Баланс в рублях`;
- `Баланс в Telegram Stars`.

Дальше можно выбрать готовую сумму или ввести свою. После этого бот предлагает оплату через Telegram Stars или CryptoBot. После успешной оплаты сумма автоматически зачисляется на выбранный баланс и сохраняется в истории пополнений.

## Файлы товаров

Стартовый каталог создаётся автоматически при первом запуске. Пути к файлам указаны в `bot/services/catalog_seed.py`.

Разложите реальные файлы и скриншоты по папкам:

- `data/assets/full/` — полные версии.
- `data/assets/demo/` — демо-версии.
- `data/assets/photos/` — фото товаров и реальные скриншоты результата.

Новые товары можно добавлять удобнее: в админ-панели фото товара и скриншот результата загружаются прямо изображениями в Telegram, а для полной и демо-версии достаточно вставить ссылку на товар. Бот сохраняет и локальный файл, и Telegram `file_id`, поэтому после обычного рестарта фото продолжают отображаться даже если локальный файл недоступен.

## Telegram Stars

Для Stars используется валюта `XTR`. Бот проверяет `pre_checkout_query`, а полный файл выдаётся только после `successful_payment`.

## CryptoBot

CryptoBot создаёт invoice через Crypto Pay API в фиатной валюте RUB. Пользователь видит сумму в рублях, а CryptoBot сам показывает эквивалент в доступных криптовалютах по актуальному курсу. Бот проверяет оплату по кнопке и дополнительно автоматически опрашивает pending-инвойсы каждые 30 секунд для пополнений баланса.

## Промокоды

В админ-панели откройте `🎟 Промокоды`. Для промокода задаются:

- код;
- процент скидки;
- срок действия по шагам: день, месяц, год, или отсутствие срока;
- лимит использований или отсутствие лимита.

Бот проверяет, что промокод существует, включён, не просрочен и не исчерпал лимит. Скидка применяется ко всем товарам и пересчитывает итоговую цену для Telegram Stars и CryptoBot.

## Отзывы

После успешной покупки бот предлагает перейти в `Мои покупки`. У каждого купленного товара есть кнопка `Оставить отзыв`: пользователь ставит оценку от 1 до 5 и пишет комментарий. Отзыв не публикуется в карточке товара, а отправляется владельцам из `ADMIN_IDS` в личные сообщения с именем покупателя, товаром, фото товара, оценкой и комментарием.

## Админ-панель

Админ-панель доступна пользователям из `ADMIN_IDS`. Через кнопку `➕ Добавить товар` бот последовательно запросит:

- название и описание;
- категорию, подкатегорию/игру, тип оптимизации и метку через кнопки;
- фото товара и скриншот результата прямой загрузкой в Telegram;
- ссылку на полную версию и демо-версию товара;
- цену и валюту цены: рубли или Telegram Stars;
- FPS до/после;
- признак экстро-оптимизации через кнопку;
- ссылку на restore-файл или инструкцию.

Через `💸 Выдать баланс` можно начислить любому Telegram ID рубли или звёзды, включая самого администратора. Через `🖼 Фото разделов` можно выбрать нужный раздел или категорию, посмотреть текущее фото, заменить его или удалить.

## Важное

Для проверки подписки бот должен быть администратором канала или иметь доступ к списку участников. Для реального магазина замените seed-файлы, фото и скриншоты на настоящие материалы.

## Бесплатный хостинг

Добавлены `Dockerfile`, `.dockerignore` и `render.yaml` для деплоя на Render Free Web Service. На Render бот работает через webhook: Telegram отправляет сообщения на `/webhook`, поэтому сервис просыпается от входящих сообщений. Локально, если `WEBHOOK_URL` не задан, бот запускается через polling.

На Render:

1. Создайте новый Web Service из GitHub-репозитория.
2. Выберите Docker/runtime из репозитория.
3. Добавьте секретные переменные окружения в Render Dashboard.
4. Не загружайте `.env` в git: он уже добавлен в `.gitignore`.

Минимально обязательные переменные:

- `BOT_TOKEN` — токен Telegram-бота.
- `CRYPTOBOT_TOKEN` — токен CryptoBot, если нужна оплата криптой.

В `render.yaml` уже прописаны несекретные значения: `ADMIN_IDS`, `REQUIRED_CHANNEL`, `REQUIRED_CHANNEL_URL`, `REVIEWS_CHANNEL_URL`, `SUPPORT_USERNAME`, `DB_PATH`, `ASSETS_DIR`, `ENABLE_TELEGRAM_STARS`, `CRYPTOBOT_API_URL`, `WEBHOOK_URL`, `WEBHOOK_PATH`. Секреты отмечены как `sync: false`, поэтому Render попросит ввести их вручную при создании сервиса из Blueprint.

Если вы поменяли имя Render-сервиса, обновите `WEBHOOK_URL` на фактический адрес сервиса, например `https://your-service.onrender.com`.

SQLite сохраняет данные в файле `DB_PATH` и переживает обычный рестарт процесса. На бесплатных хостингах контейнер и его файловая система могут очищаться при redeploy или пересоздании инстанса. Для постоянной базы на продакшене нужен persistent disk или PostgreSQL/MySQL, например внешний бесплатный PostgreSQL.
